# 文体ルール — Ubuntu×AI実践ノート

Claude Code が記事・投稿を書く際に従う文体ルールです。
必ずこのファイルを参照してから執筆を開始してください。

---

## 基本文体

- **「です・ます調」で統一する**
- 「だ・である調」は使わない
- 「〜なんですよね」「マジで便利」などの話し言葉は、ブログでは原則使わない。X・note では必要に応じて少量可
- 親しみやすさは「丁寧さ」と「具体性」で出す。硬くなりすぎない文章を目指す
- 目指すトーン：丁寧だが、少し人間味のある文章

---

## 文の長さ

- 1文は40〜80字程度を目安にする。長すぎる文は分ける
- 「〜で、〜して、〜なので」と続く複合文は切る
- 短文すぎるぶつ切りも避ける。長短を混ぜてリズムを作る

**悪い例：**
> Ubuntuをインストールして、ターミナルを開いて、コマンドを打てば環境が整うのでそのまま進めます。

**良い例：**
> Ubuntu をインストールしたら、ターミナルを開きます。コマンドを一つ打てば、環境が整います。

---

## 専門用語の扱い

- 初出の専門用語は必ず補足する
- 補足は括弧または直後の一文で行う
- 略語も同様に初出で正式名称を示す

**例：**
> CLI（コマンドラインインターフェース）を使うと、マウスを使わずに操作できます。

---

## 構成の基本

記事・段落の流れは以下を基本にする。

```
結論 → 理由 → 手順・具体例
```

- 最初に「何が分かるか」「何ができるか」を示す
- 理由や背景はその後に置く
- 手順は番号付きリストで書く

---

## 語尾のバリエーション

同じ語尾が3回以上続くと単調に見えます。以下を意識して変化をつけてください。

| 場面 | 使える語尾・表現 |
|------|----------------|
| 説明 | 〜です。／〜になります。／〜といえます。 |
| 手順 | 〜します。／〜してください。／〜しましょう。 |
| 補足 | 〜ちなみに〜。／〜なお〜。 |
| 強調 | 〜がポイントです。／ここが重要です。 |
| まとめ | 〜でした。／〜ということになります。 |
| 疑問形 | 〜ではないでしょうか。／〜と思いませんか。（多用しない） |
| 体言止め | 必要に応じて使う。多用しない。例：「これが基本の流れです。」→「基本の流れはこちら。」 |
| 疑問形（補足） | 必要に応じて使う。多用しない。記事内に2〜3回程度が目安 |

---

## 避けるべき表現

以下の表現は AI っぽさが出やすいため、使わないか言い換える。

| 使わない表現 | 言い換え例 |
|------------|-----------|
| 〜することが可能です | 〜できます |
| 〜という点において | 〜という面では |
| 〜を行うことで | 〜すると |
| 非常に重要です | 重要です／大切です |
| 〜していただけます | 〜できます |
| 〜してみてください | 〜してみましょう |
| 本記事では | この記事では |
| 上記の通り | 先ほどの通り |
| 〜となっております | 〜です |

---

## 見出しの書き方

- `##` は大見出し（セクション区切り）
- `###` は小見出し（手順のステップ、補足説明）
- 見出しは体言止めか「〜する方法」「〜の手順」の形を基本にする
- 「〜について」「〜に関して」は避ける

---

## リストの使い方

- 並列の説明は箇条書きリストを使う
- 手順は番号付きリスト（`1.` `2.` `3.`）を使う
- リスト項目は簡潔に。説明が必要な場合は項目の下に文章を添える

---

## 読者への語りかけ

- 「あなた」は使わない。「読者」「初心者の方」も多用しない
- 「〜の方は」「〜している場合は」のように状況で絞る
- 読者を直接呼びかけるより、一緒に進む感覚を大切にする（例：「〜していきます」）

---

## 既存記事との統一

- 記事を書く前に既存記事を1〜2本確認し、文体・語尾のトーンを合わせる
- 「同じ人が書いた」と感じられる一貫性を最優先にする
